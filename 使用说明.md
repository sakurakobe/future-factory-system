## 一、系统简介

### 系统是什么？

这是一个用于**现场诊断评估**的 Web 应用系统。咨询师（您）到企业现场，通过该系统：

1. **逐项核对**浙江省"未来工厂"建设导则中的 105 道评估题目
2. **记录沟通**内容和企业现状
3. **选择等级**（A-F 六级），系统自动计算得分
4. **生成报告**：一键生成符合标准格式的 Word 诊断报告

### 核心数据

| 层级 | 数量 | 说明 |
|------|------|------|
| 建设要素 | 3 个 | 赋能保障、基础支撑、应用场景 |
| 子类 | 14 个 | 如技术支撑、体系保障、智能制造等 |
| 细项 | 44 个 | 如新一代信息技术、生产计划等 |
| 题目 | 105 道 | 每道题有 A-F 六个等级描述，对应 0-5 分 |
| 满分 | 500 分 | 总分 = 105 道题得分之和 |

### 系统架构

```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│   前端界面   │  ←→   │  后端 API   │  ←→   │  SQLite数据库 │
│ React + Vite│       │  FastAPI    │       │  future_    │
│  TypeScript │       │  Python     │       │  factory.db │
└─────────────┘       └─────────────┘       └─────────────┘
   端口:3000            端口:8000            本地文件
```

## 二、环境准备

### 必要软件

| 软件 | 版本要求 | 用途 |
|------|----------|------|
| Python | 3.10+ | 运行后端服务 |
| Node.js | 18+ | 运行前端开发服务器 |

### 检查环境是否就绪

打开命令行（cmd 或 PowerShell），依次运行：

```bash
# 检查 Python
python --version
# 应该输出类似：Python 3.12.0

# 检查 Node.js
node --version
# 应该输出类似：v20.x.x

# 检查 npm
npm --version
# 应该输出类似：10.x.x
```

如果提示"不是内部或外部命令"，请先安装对应软件：
- Python: https://www.python.org/downloads/ （安装时勾选"Add to PATH"）
- Node.js: https://nodejs.org/ （下载 LTS 版本）

### 安装依赖

```bash
# 1. 进入后端目录
cd D:\code\future-factory-system\backend

# 2. 安装 Python 依赖（首次运行需要）
pip install -r requirements.txt

# 3. 进入前端目录
cd D:\code\future-factory-system\frontend

# 4. 安装 Node.js 依赖（首次运行需要）
```

## 三、快速启动

### 第一步：初始化数据库（首次运行）

```bash
cd D:\code\future-factory-system\backend
python seed/init_db.py
```

**预期输出：**
```
正在导入XLSX题库数据...
题库导入完成！
```

> ⚠️ 注意事项：
> - 此步骤只需要在**首次运行**时执行一次
> - 题库文件路径默认指向 `D:/code/文件内容/未来工厂标准诊断评估题库.xlsx`
> - 如果文件路径不同，修改 `backend/config.py` 中的 `XLSX_PATH` 变量
> - 重复执行会清除已有数据并重新导入

### 第二步：启动后端服务

```bash
cd D:\code\future-factory-system\backend
python -m uvicorn main:app --reload --port 8000
```

**预期输出：**
```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete.
```

> ✅ 看到以上信息说明后端启动成功！
> - 按 `Ctrl+C` 可以停止后端服务
> - 访问 http://localhost:8000/docs 可以查看 API 文档

### 第三步：启动前端服务

**打开一个新的命令行窗口**（不要关闭后端的那个窗口）：

```bash
cd D:\code\future-factory-system\frontend
npm run dev
```

**预期输出：**
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
```

> ✅ 看到以上信息说明前端启动成功！
> - 浏览器会自动打开 `http://localhost:3000`
> - 如果没有自动打开，手动访问 http://localhost:3000

### 第四步：开始使用

浏览器打开 http://localhost:3000，即可看到系统界面。

## 四、使用指南

### 1. 创建评估项目

1. 打开 http://localhost:3000
2. 点击页面右上角的 **"+ 新建项目"** 按钮
3. 填写以下信息：
   - **企业名称**：输入被评估企业的全称（必填）
   - **企业类型**：选择"通用"、"离散制造"或"流程制造"
   - **评估日期**：选择现场评估的起止日期（可选）
   - **评估人员**：填写参与评估的专家名单（可选）
4. 点击 **"创建项目"**，进入项目详情页面

#### 操作流程：

**步骤1：整体沟通记录**
- 在每个建设要素（如"赋能保障"）的顶部，有一个**"整体备注"**文本框
- 在与企业整体沟通某个方面时，将讨论要点记录在此
- 输入后自动保存，无需手动操作

**步骤2：逐题核对**
- 展开三级分类（方面 → 子类 → 细项），找到具体题目
- 每道题的卡片包含：
  - **题目文字**：显示评估要求
  - **沟通等级选择**：点击 A-F 按钮选择企业当前达到的等级
  - **等级描述预览**：选中等级后，自动展示该等级的详细描述
  - **企业现状**：记录企业当前的实际情况
  - **沟通内容**：记录与企业沟通的要点
  - **目标等级**：选择企业希望达到的等级

**步骤3：自动评分**
- 选择沟通等级后，系统自动计算该题得分
- 页面右上角实时显示各级别得分汇总
- 得分率用颜色标识：
  - 🟢 绿色：≥60%（良好）
  - 🟡 黄色：40%~60%（一般）
  - 🔴 红色：<40%（较差）

**步骤4：数据自动保存**
- 每次修改等级、文本内容时，数据自动发送到后端保存
- 无需手动点击保存按钮
- 关闭浏览器后重新打开，数据不会丢失

### 3. 补充信息

在"补充信息" Tab 中：
- 可以上传与评估相关的附件文件（如企业工艺流程图、车间布局图等）
- 可以添加文字补充说明

### 4. 投资计划

在"投资计划" Tab 中：
- 查看系统推荐的改造投资项目
- 按类别（必要投入/建议投入/系统已有待优化/已明确投入）管理
- 每个项目包含名称、说明、预算范围

### 5. 生成报告

1. 切换到 **"报告"** Tab
2. 点击 **"生成并下载报告"** 按钮
3. 系统自动生成 Word 格式的诊断报告并下载

**报告内容包含：**
- **概述**：企业背景、总分、得分率、雷达图数据
- **要求细项**：每个方面的详细分析（要求解读 + 企业现状 + 主要问题）
- **详细评测表**：105 道题的逐项评估结果
- **改造投入建议**：投资项目汇总表

## 五、功能详解

### 题目等级说明

每道题目有 6 个等级选项，通常对应如下：

| 等级 | 分值 | 含义 |
|------|------|------|
| A | 0 分 | 未实现/空白 |
| B | 1 分 | 初步阶段 |
| C | 2 分 | 基本实现 |
| D | 3 分 | 部分实现 |
| E | 4 分 | 较好实现 |
| F | 5 分 | 完全实现 |

> 部分题目的分值可能不是整数（如 1.67、3.33），这是根据标准设置的。

### 评分规则

```
单题得分 = 选中等级对应的分值
大类得分 = 该大类下所有题目得分之和
总分     = 赋能保障 + 基础支撑 + 应用场景（满分500分）
得分率   = 总分 / 500 × 100%
```

### 得分率参考

| 得分率 | 评估水平 |
|--------|----------|
| ≥ 80% | 未来工厂水平 |
| 60%~80% | 智能工厂水平 |
| 40%~60% | 数字化车间水平 |
| < 40% | 数字化转型初期 |

## 六、常见问题

### Q1: 启动后端报错 "No module named 'xxx'"
**解决方法：**
```bash
cd D:\code\future-factory-system\backend
pip install -r requirements.txt
```

### Q2: 前端页面显示空白或报错
**解决方法：**
1. 确认后端已启动（另一个命令行窗口运行 `uvicorn main:app`）
2. 确认前端和后端同时在运行
3. 按 F12 打开浏览器开发者工具，查看 Console 中的错误信息

### Q3: 初始化数据库报错 "XLSX 文件不存在"
**解决方法：**
1. 检查 `backend/config.py` 中的 `XLSX_PATH` 路径是否正确
2. 确认题库文件确实存在于该路径
3. 或者通过环境变量指定路径：`set XLSX_PATH=D:\xxx\xxx.xlsx`

### Q4: 如何备份数据？
**方法：** 复制 `backend/future_factory.db` 文件即可。
这是 SQLite 数据库文件，包含所有评估项目数据。

### Q5: 如何重置系统？
```bash
# 删除数据库文件
del D:\code\future-factory-system\backend\future_factory.db

# 重新初始化
cd D:\code\future-factory-system\backend
python seed/init_db.py
```

### Q6: 报告生成失败
**解决方法：**
1. 确认已安装 `python-docx` 库
2. 检查 `uploads` 目录是否有写入权限
3. 查看后端命令行窗口的错误日志
